## 三、Bug 修复记录

### Bug 1：前端登录错误提示不显示

**文件：** `frontend/src/views/LoginPage.vue`

**原因：** catch 块中 `if (e?.message || e?.length) return` 把所有带 message 的异常都静默忽略，导致后端返回的错误消息（如"用户名或密码错误"）不展示给用户。

**修复：** 改为区分表单校验错误（array）和 API 错误（message），API 错误展示后端返回的 message。

```diff
  } catch (e) {
-   // validate() 失败会 reject，但不用提示
-   if (e?.message || e?.length) return
-   message.error('登录失败，请检查网络连接')
+   // validate() 失败会 reject (array of errors)，不需要额外提示
+   if (Array.isArray(e) && e.length > 0) return
+   // API 错误（如密码错误）展示后端返回的 message
+   if (e?.message) {
+     message.error(e.message)
+   } else {
+     message.error('登录失败，请检查网络连接')
+   }
  }
```

### Bug 2：租户隔离拦截级联删除（已修复并记录）

**原因：** 删除用户时，租户行拦截器在 DELETE 语句后追加了当前用户的 `user_id`，导致 `WHERE user_id = 当前用户 AND user_id = 被删用户` 永远匹配 0 行。

**修复：** 在所有 mapper 的 `deleteByUserId` 方法上添加 `@InterceptorIgnore(tenantLine = "true")` 注解。
